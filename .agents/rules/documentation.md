# Documentation Rule

After completing ANY feature/module/significant change:
1. Update or create docs/<module-name>.md with:
   - What it does (2-3 sentences)
   - Data flow: entity → repository → usecase → API route → UI
   - Key files touched (with paths)
   - Any package used and why
2. If a decision affects future work, add it to STATE.md (if loop-engineering files exist)
3. Never end a task without this — treat it as part of "done," not optional
